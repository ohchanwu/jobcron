# Superpowers Documentation

This tree stores implementation knowledge, not general product documentation.

- Read `README.md` first and load only the active plan, specification, or
  decision needed for the current task.
- Keep unimplemented approved work in `plans/` and active or awaiting-approval
  requirements in `specs/`.
- Put concise, durable conclusions in `decisions/`.
- Move useful completed designs and implementation records to `archive/` and
  follow `archive/AGENTS.md`; do not scan the archive during ordinary work.
- Move raw briefs, command output, progress logs, and machine-local evidence to
  the ignored `.superpowers/archive/` tree after removing credentials and
  personal data.
- Prefix content files below this tree with their best-supported `YYMMDD`
  creation date. Structural `README.md` and `AGENTS.md` files are exempt.
- Update this tree's index and `docs/README.md` when adding, moving, or removing
  durable records.
- Treat every tracked file as public. Use placeholders for credentials,
  addresses, account identifiers, email addresses, and machine-specific paths.
