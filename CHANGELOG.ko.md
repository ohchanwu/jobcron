# Changelog

_[Read this in English 🇬🇧](CHANGELOG.md)_

이 프로젝트의 모든 주목할 만한 변경 사항은 이 파일에 기록됩니다.

형식은 [Keep a Changelog](https://keepachangelog.com/en/1.1.0/)를 기반으로 하며,
이 프로젝트는 [Semantic Versioning](https://semver.org/spec/v2.0.0.html)을 따릅니다.

이 changelog는 **v1.0 시점(epoch)** 에서 시작합니다 — 태깅(tagging)이 시작되기 전에
이미 배포되었던 기준선(다중 출처 스크래핑, 포털 간 중복 제거, 점수화 엔진,
일일 브리핑 UI)입니다. 그 이전의 개발 내역은 git 히스토리에 담겨 있습니다. 최초로
태깅된 릴리스는 **v1.5.0**이며, 1.0 기준선 위에 쌓인 개선 사항들을 모은 것입니다.
BYOK AI 통합은 **v2.0** 라인입니다 — 1.x 시리즈의 일부가 아닌 별도의 메이저 버전입니다.

## 릴리스를 만드는 방법 (How to cut a release)

1. `[Unreleased]`에서 항목을 빼내어 오늘 날짜로 된 새 섹션으로 옮깁니다.
   예: `## [1.0.0] - 2026-06-15`.
2. 맨 위에 비어 있는 새 `[Unreleased]` 블록을 엽니다.
3. `git tag -a vX.Y.Z -m "Release X.Y.Z"`와 `git push origin vX.Y.Z`를 실행합니다.
   GoReleaser(`.goreleaser.yml`)가 태그를 읽어
   `-ldflags "-X main.version={{ .Version }}"`를 통해 바이너리에 새겨 넣고,
   이후 `--version` CLI 플래그가 이를 표시합니다.

## [2.0.0-alpha2] - 2026-06-03

**v2.0 BYOK-AI** 라인의 2단계(Stage 2) + 하드닝(hardening)입니다. 이제 AI는 실제로
작동하는 엔드투엔드(end-to-end) 기능입니다: 프로필 폼에서 활성화하면 브리핑에 증거
인용 기반 점수 조정이 더해지며, 필요할 때마다 다시 실행할 수 있습니다. 여전히 완전한
선택 사항입니다 — 키가 설정되지 않으면 앱은 v1.5와 정확히 동일하게 동작합니다.

### 추가됨 (Added)
- **이제 AI가 사용자에게 노출됩니다.** 프로필 폼의 새로운 "AI 분석" 섹션에서 제공자
  (Anthropic / OpenAI)를 선택하고, API 키를 입력하며(데이터베이스가 아니라 로컬 0600
  파일에만 저장되고, 저장 후에는 "•••• 저장됨"으로 표시됨), "오늘 사용 / 남은 예산"이
  실시간으로 표시되는 일일 토큰 예산을 설정합니다. 앱은 시작 시점과 키를 저장하는 순간에
  제공자를 연결합니다 — 재시작이 필요 없습니다.
- **증거 인용 기반 AI 점수 조정.** AI가 켜져 있으면 각 공고는 `AI 분석` 칩 — 금색 `+N`
  또는 흐린 `−N` — 을 얻을 수 있으며, 이를 클릭하면 각 조정을 뒷받침하는 공고 속 정확한
  인용문을 볼 수 있습니다(또는 목표에는 필요했지만 공고에 없는 항목에 대해서는 코드로
  검증된 "관련 언급 없음"을 표시). 표시되는 모든 조정은 실제 인용으로 뒷받침되며,
  근거 없는 것은 나타나지 않습니다.
- **재평가 (re-rate).** 페이지별 버튼(브리핑 / 관심 공고 / 북마크)으로, 화면에 보이는
  행들을 당신의 목표 기준으로 다시 점수화하며 진행 상황을 스트리밍합니다. 현재 목표
  기준으로 아직 분석되지 않은 행에만 토큰을 사용하므로, 다시 누르면 작업을 처음부터 다시
  하지 않고 중단된 지점부터 이어 갑니다. 키가 설정되지 않았을 때는 완전히 숨겨지며,
  일부 행이 더 오래된 프로필 기준으로 점수화되었을 때는 그 개수를 표시합니다.
- **롤링(rolling) 일일 토큰 원장(ledger) + 상한.** 실행당 상한과 일일 토큰 상한
  (기본값 약 1M/일)이 지출을 제한합니다. 예산이 소진되면 브리핑은 오류가 아니라 차분한
  안내와 함께 일반 점수로 마무리됩니다.

### 변경됨 (Changed)
- 공고의 "(이전 프로필 기준)" 오래된(stale) AI 칩은 합계에 계속 포함된 채 흐리게
  표시됩니다. 따라서 목표를 편집해도, 점수가 이제 최신이 아니라는 사실을 알리지 않고
  조용히 바꾸는 일은 없습니다.

### 보안 (Security)
- AI는 신뢰할 수 없는(untrusted) 대상으로 취급됩니다: 모델을 탈취하려는 공고(숨겨진
  "지시를 무시하라 / 키를 덤프하라" 텍스트)는 그 출력이 JSON 게이트와 인용 게이트에서
  거부되어, 어떤 점수도 적용하지 못하며, 키를 빼낼 수도 없습니다(키는 애초에 프롬프트에
  들어가지 않으며, 외부 통신(egress)은 단 하나의 제공자 호스트로 고정됩니다).

### 알려진 이슈 (Known issues)
- AI가 켜져 있을 때 `인턴`(인턴십) 공고가 `신입` 칩을 잃을 수 있습니다: 모델이 때때로
  엔트리 레벨 인턴 직무를 경력이 필요한 것으로 읽고, 점수가 출처의 올바른 신입 플래그보다
  그 판단을 더 신뢰합니다. 수정 대기 중입니다.
- 실제 제공자 통합 테스트는 포함되어 있으나 선택 실행(opt-in)이며, 이번 릴리스에서는
  실행되지 않았습니다(사용 가능한 API 키 없음).

## [2.0.0-alpha] - 2026-06-02

**v2.0 BYOK-AI** 라인이 여기서 시작됩니다(1.x의 일부가 아닌 별도의 메이저 버전).
아래의 1단계(Stage 1)는 그 토대입니다 — 커밋되었으나 **잠들어 있습니다(dormant)**: 아직
키 입력 UI가 없으므로(이후 단계), 제공자가 설정되기 전까지 실행 중인 앱은 v1.5와 정확히
동일하게 동작합니다.

### 추가됨 (Added)
- **당신의 키를 직접 사용하는 AI 제공자 레이어(`internal/ai`).** 직접 작성한 Anthropic 및
  OpenAI HTTP 클라이언트(순수 Go 단일 바이너리 빌드를 유지하기 위해 SDK 미사용)를 갖춘
  제공자 인터페이스, 단일 호스트 외부 통신 고정(one-host egress pin), 0600 권한의
  `ai_keys.json` 키 저장소, 그리고 오프라인 테스트 모음을 위한 스텁(stub) 제공자.
- **1단계 추출(extraction) + 캐시.** AI가 활성화되면 새 공고마다 그 경력/학력 요건을
  모델이 읽어 새로운 `ai_extractions` 테이블(마이그레이션 `0008`)에 캐싱합니다.
  점수화는 정규식(regex) 휴리스틱보다 이 캐시된 AI 사실을 우선합니다 — 그래서 본문에
  "경력 5년 이상"이라고 적혀 있지만 실제로는 신입 친화적인 공고가 0점이 아니라 온전한
  신입 점수를 받습니다. AI 처리가 실패하면 기존 정규식 점수화로 조용히 폴백(fallback)합니다.
- **프로필 AI 목표 필드.** 새로운 "AI 분석 (선택)" 섹션 아래의 네 개의 선택적 자유 서술형
  필드(좋아하는 업무, 피하고 싶은 업무, 단기/장기 목표)로, 곧 추가될 AI 분석을 위한
  것입니다.

### 제거됨 (Removed)
- **필수 키워드 (must-have keywords).** 필수 항목 목록이 사라졌습니다. 대신 제외
  키워드(하나라도 일치하면 공고 제외)를 사용하세요. 과거의 필수 항목은 목표 필드나
  제외 키워드로 다시 입력하세요.

### 변경됨 (Changed)
- **점수가 `[0, 100]`으로 고정(clamp)됩니다**(하한선이 추가됨). 그리고 점수 세부 내역은
  부호가 붙은 델타(signed delta)로 렌더링됩니다. AI가 꺼져 있는 동안 기존 점수에는 영향이
  없습니다.

## [1.5.0] - 2026-05-29

1.0 기준선 위에 누적된 개선 사항들로, 최초로 태깅된 릴리스로 정리되었습니다. 여기 있는
모든 것은 키워드 매칭 기반 점수화와 UI입니다 — LLM은 없습니다. AI는 v2.0입니다.

### 추가됨 (Added)
- **관심 공고 뷰.** 기존의 "전체 공고" 페이지가 이제 선별된 목록이 되었습니다. 최저 점수에
  미달하는 공고는 데이 리스트(day list)를 어지럽히는 대신 "관심 밖으로 분류된 공고"
  섹션으로 접혀 들어가며, 이는 일일 브리핑과 동일한 방식입니다. (URL은 `/archive`로
  유지됩니다.)
- **관심 없음 수동 숨김(manual mute).** 클릭 한 번으로 공고를 브리핑과 관심 공고에서
  숨깁니다. 북마크되었지만 숨겨진 공고는 북마크 페이지에 남아 있으며, 프로필 설정의 숨김
  해제 목록에서 언제든 되돌릴 수 있습니다.
- **항목별 점수 가중치.** 경력 적합도와 연봉 적합도의 비중을 설정합니다. 기본값은 기존의
  25/10 점수화를 보존합니다. 프로필 폼은 가중치를 바꿀 때 그로부터 도출되는 근소한
  차이(near-miss) 점수와 모호한 연봉(ambiguous-salary) 점수를 미리 보여줍니다.
- **최저 점수 임계값.** 점수가 낮은 공고를 일일 브리핑에서 숨깁니다(기본값 40; 0으로
  설정하면 모두 표시).
- **테마 전환.** 라이트 / 다크 / 자동이며, 모니터·해·달 아이콘이 함께 표시됩니다.
- **파비콘(Favicon).** 금색 일출 마크 — 다중 해상도 ICO 폴백을 갖춘 SVG입니다.

### 변경됨 (Changed)
- 지역 매칭이 강남을 강남구(및 유사한 자치구 형태)로 취급하므로, 더 많은 서울 공고가
  지역 보너스를 받습니다.

### 수정됨 (Fixed)
- 데모데이: IT/SWE 필터가 모든 직무 버킷(bucket)에 적용되며, 프로그래머 직함의 직무가
  버려지지 않고 유지됩니다.
- 오래된 공고 정리(stale-posting sweep): 3일 기준 창(window)이 이제 경계 테스트로
  고정되었습니다(딱 미만은 살아남고, 딱 초과는 다음 스크랩 때 제거됨).
